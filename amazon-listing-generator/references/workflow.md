# Amazon Listing Workflow

## Goal

Convert product facts, keyword intent, competitor observations, and buyer pain points into Amazon listing content that works for both shoppers and Amazon systems.

## Workflow

1. Normalize product facts.
2. Identify missing blocking specs.
3. Prioritize the main search terms and buyer pain points.
4. Write mobile-readable title and bullet points.
5. Generate Search Terms with minimal repeated words.
6. Generate Product Description in Amazon-compatible HTML format.
7. Create image, A+, and video briefs when requested.
8. Validate with Rufus-style buyer questions.
9. Run listing self-check.

## Quality Rules

- The first screen matters most: title, main image, first 3 images, first 2 bullets, and A+ first module.
- Keep the strongest differentiator near the front.
- Make specs explicit when the user provides them.
- Do not turn competitor data or market assumptions into consumer-facing claims.
- Keep front-end copy, backend terms, image brief, and Rufus answers consistent.
